# Internet Banking Services — Knowledge Base

## Overview
This category covers Bank Muscat's internet banking platform, Soft Token app, Secure Token hardware device, Corporate Online Banking (Corp-Online), and related authentication/access issues. Also covers WPS salary registration issues.

## Products & Services

### Internet Banking (ib.bankmuscat.com)
Full-feature online banking for personal and business customers. Requires User ID, password, and OTP (via SMS or token).

### Soft Token (Mobile App)
Bank Muscat's mobile application for two-factor authentication. Generates time-based OTPs. Separate from the mBanking app.
- **Common error**: "Application under maintenance" message when entering PIN.
- **Resolution**: Advise Customer to uninstall and reinstall the app, then re-register. This resolves most maintenance state errors.

### Secure Token (Hardware Device)
Physical OTP device issued at branch. Used for internet banking authentication.
- Issued at branch; must be activated after collection.
- Issues: Token not received (Branch Delay), Token not working/active (Operational Issue or System Error), Token showing SYS error.

### Corporate Online Banking (Corp-Online)
Dedicated internet banking portal for corporate customers at corp-online.meethaq.om. Has its own FAQ and help files.

### WPS (Wage Protection System) Registration
- WPS is Oman's government-mandated salary protection system. Employers must pay salaries via approved channels.
- WPS registration for salary transfers requires the **Sanad portal** (Ministry of Manpower). Branch staff cannot process WPS registrations directly.
- **Common issue**: Customers/employers directed to Sanad office unnecessarily, or branch staff incorrectly handle WPS registration.
- **Root Cause**: Operational Issue — process misalignment; branch staff lack proper training or system access.

## Common Issues & Root Causes

### Soft Token Showing "Application Under Maintenance"
- **Root Cause**: Operational Issue — infrastructure failure or unscheduled maintenance with inadequate customer communication.
- **Known pattern**: Occurs when the soft token backend service is under maintenance or experiencing instability.
- **Resolution**: Advise Customer to re-install app and re-register. If systemic, escalate to IT team.

### Internet Banking System Unavailable / Login Failure
- **Root Cause**: System Error — platform unavailability. Could be planned maintenance or unplanned outage.
- **Resolution**: Rectify Technical Error; inform customer of alternative channels (branch, ATM, phone 24795555).

### Secure Token Not Received
- **Root Cause**: Branch Delay — token was issued but not handed to customer.
- **Resolution**: Handover The Token To The Customer.

### Secure Token Not Working / Disabled / SYS Error
- **Root Cause**: Operational Issue or System Error — token not linked to account, or activation failed.
- **Resolution**: Activate the Token; Check if the Token is Linked and Activated; Branch needs to check Customer Application.

### Token Not Linked to Account
- **Root Cause**: Operational Issue or System Error — token was given to customer but the linking step in the system was not completed.
- **Resolution**: Activate the Token; verify linking in system.

### Online User Registration Not Processed
- **Root Cause**: Operational Issue or System Error.
- **Resolution**: Contact Customer And Inform Accordingly; Rectify Technical Error.

### Not Receiving OTP
- **Root Cause**: Operational Issue (wrong phone number on file) or System Error (SMS gateway failure).
- **Resolution**: Contact Customer And Inform Accordingly; Rectify Technical Issue; verify registered mobile number.

### Transaction Access Not Given to Corporate User
- **Root Cause**: System Error — access rights not properly configured in Corp-Online.
- **Resolution**: Contact Customer And Inform Accordingly; Rectify Technical Error.

## Key Operational Facts
- SWIFT code published on the website but poorly indexed — common source of customer inquiries.
- The bank's internet banking and Soft Token app are separate products from mBanking.
- Soft Token reinstallation is the primary self-service fix for most token authentication errors.
- WPS-related issues always route through Sanad portal — branch staff should not attempt to process these directly.
- Al Khuwair branch was specifically mentioned as needing refresher training on WPS procedures.
